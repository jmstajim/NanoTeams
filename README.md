## Why NanoTeams

NanoTeams was designed from the ground up for **local LLMs**.

The goal is simple: make LLMs work **as fast and efficiently as possible** in real tasks.

## Features

- **Built for local LLMs**  
  NanoTeams was specifically made for local models and tuned for practical multi-role workflows.

- **LM Studio's Stateful Chats**  
  No need to reprocess the full conversation every turn.

- **Quick context capture**  
  Grab files, images, and text from any app with hotkeys.

- **Built-in tools**  
  Includes search, file operations, Git, `xcodebuild`, and team tools like:
  - `ask_teammate`
  - `request_team_meeting`
  - `request_changes`

- **Work in folder**  
  File access is limited to the chosen folder.

- **Different LLMs for different roles**  
  Use multiple models for different team members in a single run.

- **Separate vision model**  
  Use a dedicated model for image understanding.

- **Custom teams**  
  Create your own teams, then import or export them.

- **No telemetry**  
  NanoTeams doesn’t send your data anywhere, and debug logs are off by default.

## Built-in Teams

- **Personal Assistant**  
  A conversational helper for any task.

- **FAANG Team**  
  A full product pipeline from “I want ummm...” to release.

- **Engineering Team**  
  A lean engineering pipeline.

- **Startup**  
  One engineer and that startup energy.

- **Quest Party**  
  Five specialists build the world, characters, and encounters, then the Quest Master leads an interactive story where you are the hero.

- **Discussion Club**  
  Five strong personalities argue, debate, and challenge each other in a lively discussion.
